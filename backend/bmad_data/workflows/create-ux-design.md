# Create UX Design Workflow (14 Steps)

## Overview
Create a comprehensive UX specification through systematic user-centered design. Every design decision must trace back to user needs and business goals.

## Prerequisites
- PRD document (strongly recommended)
- Project brief or equivalent context

## Step-by-Step Instructions

### Step 1: Init — Input Discovery
**Goal:** Gather all available context for UX design.

- Load and review:
  - PRD (user journeys, functional requirements)
  - Project brief (problem, users, vision)
  - Any existing designs or mockups
  - Competitive analysis or benchmarks
- Identify design constraints and opportunities
- Summarize the UX challenge

### Step 2: User Research
**Goal:** Deep understanding of users, personas, and mental models.

- Review/create user personas with:
  - Demographics and context
  - Goals and motivations
  - Frustrations and pain points
  - Technical proficiency
  - Behavioral patterns
- Map mental models: How do users think about this problem space?
- Identify user expectations based on existing alternatives

### Step 3: Information Architecture
**Goal:** Define content structure and navigation.

- Organize content into logical groups
- Define navigation hierarchy (max 3 levels deep)
- Create sitemap/screen inventory
- Define naming conventions for navigation items
- Consider search and discovery patterns

### Step 4: User Flows
**Goal:** Map core task flows and decision points.

- For each primary user journey:
  - Entry point → key actions → completion
  - Decision points and branches
  - Error recovery paths
  - Shortcuts for expert users
- Identify flow dependencies and prerequisites
- Optimize for minimal steps to completion

### Step 5: Wireframe Concepts
**Goal:** Low-fidelity layout exploration.

- Describe screen layouts in structured text
- For each key screen:
  - Purpose and context
  - Content hierarchy
  - Primary and secondary actions
  - Layout structure (header, content areas, navigation)
- Explore 2-3 layout alternatives for critical screens
- Select preferred direction with rationale

### Step 6: Interaction Patterns
**Goal:** Define micro-interactions and behaviors.

- Document interaction patterns:
  - Form interactions (validation, auto-complete, inline editing)
  - Navigation transitions
  - Data loading and refresh patterns
  - Drag-and-drop behaviors
  - Keyboard shortcuts
  - Gesture support (mobile)
- Reference established pattern libraries where applicable

### Step 7: Component Library
**Goal:** Define reusable UI component specifications.

- Identify recurring UI patterns
- For each component:
  - Purpose and usage context
  - States (default, hover, active, disabled, loading, error)
  - Variants and sizes
  - Responsive behavior
- Group by category (navigation, input, display, feedback, layout)

### Step 8: Responsive Strategy
**Goal:** Multi-device adaptation approach.

- Define breakpoints and target devices
- For each major layout:
  - Desktop adaptation
  - Tablet adaptation
  - Mobile adaptation
- Identify mobile-first vs desktop-first components
- Define touch-target sizes and spacing

### Step 9: Accessibility
**Goal:** WCAG compliance and inclusive design.

- Define target WCAG level (minimum AA)
- Address:
  - Color contrast requirements
  - Screen reader support
  - Keyboard navigation
  - Focus management
  - ARIA landmarks and labels
  - Alternative text for images
  - Reduced motion support
- Document accessibility testing approach

### Step 10: Error Handling UX
**Goal:** Design error states and recovery flows.

- For each error category:
  - Visual presentation
  - Error message copy (clear, actionable)
  - Recovery options
  - Prevention strategies
- Types: validation errors, system errors, network errors, permission errors
- Design for graceful degradation

  - No-data states
  - Search with no results
  - Onboarding guidance
- Loading states:
  - Skeleton screens
  - Progress indicators
  - Optimistic updates
  - Background loading patterns
