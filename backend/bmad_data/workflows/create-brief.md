# Create Project Brief Workflow

## Overview
Guide the user through creating a comprehensive project brief that captures the essence of their idea, the problem it solves, who it serves, and what makes it different.

## Step-by-Step Instructions

### Step 1: Understand the Project Idea
**Goal:** Get the user to articulate their idea clearly.

- Ask the user to describe their project idea in their own words
- Listen for: the core concept, the spark of inspiration, any existing work
- Ask clarifying questions:
  - "What inspired this idea?"
  - "Have you seen anything similar? What would you do differently?"
  - "Is this greenfield (new) or brownfield (existing project)?"
- For brownfield: discover existing documentation, codebase, and constraints
- **Do NOT generate content yet** — this is pure discovery

**Menu:**
- **[A] Advanced Elicitation** — Use Socratic questioning or first-principles analysis to deepen understanding
- **[P] Party Mode** — Bring in other perspectives for collaborative brainstorming
- **[C] Continue** — Proceed to next step

### Step 2: Identify Target Users
**Goal:** Define who benefits from this product.

- Help the user identify primary and secondary user personas
- For each persona, explore:
  - Demographics and context
  - Pain points and frustrations
  - Current alternatives they use
  - What would delight them
- Validate: "Are these the people whose lives this product will improve?"

**Menu:** [A] Advanced Elicitation | [P] Party Mode | [C] Continue

### Step 3: Define Core Problem
**Goal:** Crystallize the problem being solved.

- Guide the user to articulate the problem in one clear sentence
- Explore:
  - Why does this problem matter?
  - What happens if it's NOT solved?
  - How are people solving it today? (alternatives and workarounds)
  - What's the cost of the status quo?
- Test: Can you explain this problem to a stranger in 30 seconds?

**Menu:** [A] Advanced Elicitation | [P] Party Mode | [C] Continue

### Step 4: Outline Key Features & Differentiator
**Goal:** Identify what makes this solution unique.

- List the core features/capabilities (keep it high-level)
- For each feature, ask: "Is this a must-have or nice-to-have?"
- Identify the key differentiator:
  - "If you could only tell someone ONE thing that makes this different, what would it be?"
- Define success criteria at a high level

**Menu:** [A] Advanced Elicitation | [P] Party Mode | [C] Continue

### Step 5: Generate Brief
**Goal:** Compile all discoveries into a structured project brief document.

- Use the product-brief template
- Sections to populate:
  - **Executive Summary** — 2-3 sentence elevator pitch
  - **The Problem** — Core problem statement with context
  - **The Solution** — High-level solution description
  - **What Makes This Different** — Key differentiator
  - **Who This Serves** — Target user personas
  - **Success Criteria** — How we know this succeeded
  - **Scope** — What's in and out of scope
  - **Vision** — Where this goes long-term
- Present the draft to the user for review
- Iterate based on feedback
- Save as project deliverable

**Menu:** [A] Advanced Elicitation | [P] Party Mode | [C] Continue
